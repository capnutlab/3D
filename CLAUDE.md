# 3Dモデリング作業ルール（このフォルダ）

このフォルダは 3Dモデル制作の作業場です。Blender + BlenderMCP を使い、Claude（私）が
コードで土台や構造を作り、ユーザー（あなた）がブラシで仕上げる、という協働を基本とします。
**このファイルのルールに沿って毎回進めること。**

---

## 0. 最重要ルール：作る前に「方向性」を議論する

**いきなりモデリングを始めない。** まず下記を確認・合意してから手を動かす。

1. **何を作るか**（対象物、用途：ゲーム/フィギュア印刷/アニメ/映像/習作 など）
2. **スタイル**（リアル / アニメ調 / チビ / ローポリ / スタイライズ）
3. **参照素材**の有無（画像 / 4面図 / 既存モデル / URL / 文章だけ）
4. **ゴール品質と手間の許容度**（ざっくり試作か、作り込むか）
5. **出力形式**（.blend / glb / fbx / obj）と用途先

→ 要件は `AskUserQuestion` でまとめて聞く（小出しにしない）。
→ 方向性が固まってから「§2 手段の選び方」で作り方を決める。

---

## 1. プロジェクト開始フロー

1. §0 で方向性を合意
2. §2 で手段（既製モデル / 自作 / AI / 各サイト）を選定
3. 参照があれば §6 の参照リグを組む
4. **ブロックアウト（粗い塊）→ スクショで確認 → 詳細化** の反復で進める
5. 節目ごとに方向性とズレがないか確認する

> 大事な前提：コードによる自作は「土台・構造」までは速いが、下絵そっくりの作り込みには
> 限界がある（私はブラシでのスカルプトや手描きができない）。**完成度を求めるなら
> 既製モデル流用やユーザーのスカルプトを早めに併用する。**

## 3. 役割分担（🤖私 / 🧑あなた）

- 🤖 得意：Pythonでの生成・変形、数値的な整列/対称/ミラー、モディファイア、
  シェーダー/マテリアル、UV自動展開、リグ生成＋自動ウェイト、ライティング、レンダー、

- 🤖 不可：ブラシでのスカルプト、見ながらの直感的な頂点操作、手描きテクスチャ、
  アプリ/アドオンのUI操作、APIキー登録。
- 🧑 得意：スカルプト、毛流れ・シワ等の仕上げ、テクスチャペイント、表情/シェイプキー、
  見た目の最終判断、外部アプリ（VRoid等）操作。

**基本ループ：私が土台を作る → あなたがブラシで仕上げる。**

## 4. モデリングの標準工程

参照セットアップ → ブロックアウト → リトポ → パーツ分離 → 形状追い込み →
UV展開 → マテリアル/テクスチャ → リギング → ライティング/レンダー

各工程の後に必ずスクショで確認し、ズレていれば前段に戻る。

---

## 5. Blender + MCP 技術メモ（ハマりどころ）

- **接続確認**：セッション開始時に `get_scene_info` で疎通確認。各統合は `*_status` で確認。
  - PolyHaven=有効。Sketchfab / Hyper3D Rodin / Hunyuan3D=既定で**無効**（チェックON＋接続再起動で有効化、要APIキー/無料トライアル）。
- 各MCPツールは `user_prompt` 引数が必須のことが多い。
- **Blender UIが日本語化**されている → シェーダーノード名もローカライズ（例：`プリンシプルBSDF`）。
  `nodes.get("Principled BSDF")` は None になる。**必ず `n.type=='BSDF_PRINCIPLED'` で取得**。
  入力名（`"Base Color"` 等）は英語キーでOK。
- 表示確認は **SOLID + `shading.color_type='MATERIAL'`（diffuse_color）** か **RENDERED**。
  MATERIALプレビューはHDRIで**白飛びしやすい**。レンダーは AgX + 露出0 で。
- `bpy.ops` が不安定なときは **bmeshで直接構築** or `bpy.context.temp_override(...)`。
- **QuadriFlow**（自動リトポ）は「多様体かつ法線一貫」が必須。
  失敗時は `remove_doubles` + `normals_make_consistent` で前処理してから再実行。
- 有機的ベースは**スキンモディファイアより「楕円球を多数配置＋ボクセルリメッシュ」が安定**
  （スキンは断面の向きが不安定）。
- Empty Image の `empty_display_size` は**長辺**を指定する。
- 既存デモ物（Cube類・退避した剣 `wooden_handle_saber`）は +X20 付近に退避中。

## 6. 参照（下絵）リグの作り方

- 4面図を Empty Image で前/後/左右に配置（`Reference` コレクション、選択ロック・半透明）。
- スケール：図形がキャンバス縦の約96%なら `empty_display_size≈1.8`、中心zを調整して足元z=0。
- **正面=-Y**（正面図を-Y側から見る配置）。つま先も-Y。
- 側面図の前後向きは要確認（顔・つま先が-Yを向くように。逆なら `scale.x=-1` で水平反転）。
- 採寸はピクセル解析（白背景からの差で図形判定）でプロポーションを数値化できる。

---

## 7. ファイル運用

- 作業ファイルは `.blend` で保存。書き出しは用途別（glb/fbx/obj）。
- パーツはオブジェクト分離＋専用マテリアル。`Character` などコレクションで整理。
- 大きな変形の前に複製バックアップ（例：`*_blob`）を非表示で残す。

---

## 8. セッション開始チェックリスト

1. `get_scene_info` で Blender 疎通確認
2. 必要な統合の `*_status` 確認（足りなければユーザーに有効化依頼）
3. §0 の方向性確認から開始
